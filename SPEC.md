# SPEC вҖ” Karttamaster SyГ¶teMTB 2026

## В§G Goal

SyГ¶teMTB 2026 merkintГӨtyГ¶kalu вҖ” suunnittelu, kenttГӨtyГ¶, purku yhdessГӨ sovelluksessa. Korvaa paperinen kartta. Kaksi roolia: jГӨrjestГӨjГӨ (suunnittelu + tilannekuva) ja talkoolainen (kenttГӨtyГ¶). Kaikki nГӨkevГӨt kaikkien statuksen.

## В§C Constraints

- Stack: Vite + TypeScript + Leaflet + Bun. Ei frameworkkia.
- Scope: SyГ¶teMTB 2026, yksi tapahtuma, 4 reittiГӨ. Ei multi-event.
- Deploy: Fly.io, Docker, nginx. Basic auth tГӨllГӨ hetkellГӨ.
- Offline: valinnainen, PWA myГ¶hemmin.
- Ei PDF-tulostusta.
- Testit: Vitest (unit) + Playwright (e2e).

## В§I Interfaces

- `public/route-*.gpx` вҖ” 4 GPX-tiedostoa (35km, 55km + 2 puuttuu?)
- `localStorage['karttamaster-layer']` вҖ” tile-layer preference
- `localStorage['karttamaster-markers']` вҖ” marker data: `{ version: 1, markers: SignMarker[], lastSyncAt?: string }`
- `GET /api/markers?event=<id>` вҖ” hae kaikki merkit tapahtumalle
- `PUT /api/markers/:id` вҖ” luo tai pГӨivitГӨ merkki (upsert)
- `DELETE /api/markers/:id` вҖ” poista merkki
- `GET /api/events/:id` вҖ” tapahtuman metatiedot (GPX-polut jne.)
- `POST /api/segments` вҖ” luo/upsert segmentti (jГӨrjestГӨjГӨ+)
- `PUT /api/segments/:id` вҖ” pГӨivitГӨ segmentti (jГӨrjestГӨjГӨ+)
- `DELETE /api/segments/:id` вҖ” poista segmentti (jГӨrjestГӨjГӨ+)
- `GET /api/segments/by-code/:code` вҖ” talkoolaisen oma pГӨtkГӨ (auth vaaditaan)
- Leaflet tile APIs: MML Taustakartta, MML Maastokartta, OSM
- `VISION.md` вҖ” product vision, luetaan suunnittelun ja testauksen referenssinГӨ
